# Спека: Claude Usage Tracker (обзор продукта)

> Обзорная спека: что продукт обещает пользователю. Правила жанра —
> [README.md](README.md). Детальная спека подсистемы задач —
> [tasks.md](tasks.md).

## §1. Назначение

Windows-приложение в системном трее: в реальном времени показывает
лимиты расхода Claude, прогнозирует их исчерпание, ведёт локальную
аналитику эффективности Claude Code и встроенный менеджер задач,
синхронизированный с сессиями Claude Code. Работает локально; общается
только с claude.ai и status.claude.com.

## §2. Мониторинг лимитов

- Поллинг usage API claude.ai по sessionKey + organization UUID;
  интервал настраивается (10–300 с, по умолчанию 60).
- Показываемые лимиты: 5-часовое окно, недельное окно, плюс недельные
  Opus/Sonnet-окна, когда план их отдаёт.
- План аккаунта (Pro / Max 5× / Max 20×) **выводится из состава окон**,
  которые вернул API, а не задаётся пользователем. План и окно — разные
  вещи: смена плана не меняет семантику окон.
- Показываются также платные надстройки, когда присутствуют: extra
  usage (pay-as-you-go) и prepaid-баланс.
- До сброса окна идёт обратный отсчёт («Resets in 4h 8m (Today
  14:10)»), даты локализованы.
- Уровни расхода подсвечиваются цветом; пороги настраиваются отдельно
  для сессионного и недельного окна.

## §3. Форекаст и бюджет

- Прогноз исчерпания каждого окна строится от скользящего среднего
  темпа расхода (окно усреднения настраивается) с индикатором темпа
  «ok / warn».
- Карточка 5-часового окна дополнительно реагирует на всплеск: при
  резком росте расхода прогноз пересчитывается по недавнему темпу, а не
  только по среднему.
- Дневной бюджет (опционально): рекомендованная сумма равномерно
  размазывает остаток недельного лимита до сброса — в процентах лимита,
  либо в долларах, когда включена аналитика Claude Code.

## §4. Уведомления

- Пороговые и форекаст-алерты, настраиваемые по окну и типу алерта.
- Quiet hours и временный mute/фокус.
- Мониторинг status.claude.com: уведомление при инциденте или деградации.
- Отдельные уведомления менеджера задач — при переходе задачи в review
  или done.

## §5. Окна

| Окно | Поведение |
|---|---|
| tray flyout (main) | основная панель у трея; можно закрепить |
| mini | компактный always-on-top виджет; опционально CPU/RAM системы |
| analytics | аналитика эффективности (opt-in) |
| tasks | менеджер задач: Kanban + граф (спека — [tasks.md](tasks.md)) |
| settings | настройки, вкладками |

Приложение живёт в трее и не занимает таскбар.

## §6. Аналитика эффективности (opt-in)

Питается локальными данными Claude Code (транскрипты, статистика):
кэш-эффективность, качество сессий, продуктивность/ROI, разбивки по
моделям и проектам, детект аномальных сессий, тренды и цели (стоимость
часа, доля ошибок), cold-rewrites с разбивкой по причинам, стоимость
задач (см. tasks §12), экспорт JSON. Наружу данные не уходят.

## §7. Автостарт сессии

По сбросу 5-часового окна приложение (если включено) создаёт чат-пинг в
изолированном проекте «Usage Tracker - Auto Session», запуская новое
окно; при неудаче — повторяет. Обычные проекты не засоряются.

## §8. Обновления и релизы

- Автообновление через updater: манифест latest.json из GitHub
  Releases, артефакты подписаны (minisign), установка passive.
- Релиз запускается из UI: бамп версии (`npm run bump`) → тег →
  release-workflow. master закрыт для прямых пушей — только PR.
- Три изолированных профиля (dev / preview / prod) с раздельными
  data-каталогами и identifier'ами; dev-запуск — только `npm run
  app:dev` (обычный `tauri dev` писал бы в продовые данные).

## §9. Интеграция с Claude Code

- Из настроек ставится связка «CLI + хуки»: бандл-копия `cli.mjs`
  подключается в `~/.claude/settings.json` как SessionStart-хук
  (контекст доски), Stop-хук (guard handoff-батона) и plan-хуки
  (ритуал plan mode). Поведение каналов — tasks §9–§10.
- Каждый модуль `scripts/cli/*.mjs` обязан быть перечислен в
  bundle.resources — установленная сборка, в отличие от dev, не находит
  неперечисленные модули.

## §10. Хранение и приватность

Сессионный ключ — локально в Tauri Store; данные приложения — в
`%APPDATA%\com.claude-usage-tracker.app\` (в т.ч. todos.json);
сторонним серверам ничего не отправляется.
